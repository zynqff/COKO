# ЕГЭ Томск — нативное Android-приложение для ЦОКО

Приложение на **Kotlin + Jetpack Compose**, которое логинится на личный кабинет
ЦОКО Томской области (`coko.tomsk.ru/exam2026`) по фамилии + серии/номеру паспорта,
парсит реальный HTML страниц сайта (через **Jsoup**) и показывает расписание
экзаменов и результаты в удобном виде. Данные для входа хранятся **зашифрованно
на устройстве** (Android Keystore + EncryptedSharedPreferences) — при следующих
запусках вход происходит автоматически, без повторного ввода.

## ✅ Что подтверждено по реальному HTML сайта (структура верна)

Парсер основан на реальных страницах `default.aspx`, `personal_data.aspx`,
`res_exams.aspx`, которые мне прислали. Это означает следующие вещи **точно
работают**:

- **Вход**: поля формы `ctl00$LastName`, `ctl00$Seria`, `ctl00$Number`,
  кнопка `ctl00$LoginButton` со значением `"Выполнить вход"`, постится на
  `default.aspx`.
- **Признак успешного входа**: появляется блок `#ctl00_LoggedUserPanel`
  с фамилией в `#ctl00_lbLastName`.
- **Профиль**: ФИО/класс/школа — из `#ctl00_ContentPlaceHolder1_lbFIO` /
  `lbClass` / `lbSchool` на `personal_data.aspx`.
- **Расписание экзаменов** (с ППЭ/аудиторией) — таблица
  `#ctl00_ContentPlaceHolder1_ChExams` на `personal_data.aspx`.
- **Результаты с баллами** — таблица `#ctl00_ContentPlaceHolder1_ResExams`
  на `res_exams.aspx`.

## ⚠️ Единственное, что осталось неточным — детализация по заданиям

Кнопка **"Просмотреть"** у результата — это не отдельная страница, а
**AJAX-постбэк** (`__doPostBack('ctl00$ContentPlaceHolder1$lb_<guid>','')`)
внутри `res_exams.aspx`, который подгружает данные в таблицу
`#ctl00_ContentPlaceHolder1_DetailsTable`. В присланном HTML эта таблица
была **пустой** (кнопку не нажимали), поэтому я не видел, как именно выглядит
разбивка по заданиям (часть 1 / часть 2 / критерии К1, К2...).

Приложение технически умеет выполнять этот постбэк
(`CokoWebClient.openResultDetail()`), и если таблица окажется пустой —
покажет вежливое сообщение, а не упадёт с ошибкой.

**Чтобы это доделать:** нажми "Просмотреть" у любого результата на сайте
в браузере (с открытыми DevTools → Network, либо просто "Просмотр кода
страницы" после клика) и пришли мне получившийся HTML — поправлю
`ResultDetailViewModel.parseDetail()` без изменения остальной архитектуры.

## Структура проекта

```
app/src/main/java/ru/coko/ege/
├── data/
│   ├── local/SecureCredentialsStore.kt   — зашифрованное хранилище паспортных данных
│   ├── remote/CokoWebClient.kt           — HTTP-логин + постбэки (ViewState/cookies)
│   ├── remote/CokoHtmlParser.kt          — Jsoup-парсинг реальных страниц сайта
│   └── repository/CokoRepository.kt      — объединяет расписание + результаты
├── domain/model/                         — модели данных (ExamCard, StudentProfile, ...)
└── presentation/
    ├── splash/    — сплеш-экран с автологином по сохранённым данным
    ├── login/     — экран входа
    ├── privacy/   — политика конфиденциальности
    ├── main/      — Bottom Navigation (Главная/Результаты/Помощь/Профиль)
    ├── results/   — детализация результата по заданиям (через AJAX-постбэк)
    ├── help/      — апелляции (аккордеон) + форма поддержки
    └── profile/   — профиль, настройки, выход из аккаунта
```

## Как собрать

1. Установи **Android Studio** (Ladybug или новее).
2. Открой папку `CokoEge` через "Open" в Android Studio — она сама подтянет
   Gradle (wrapper уже включён в архив) и все зависимости.
3. Подключи телефон или эмулятор (Android 7.0+, API 24+) и нажми **Run**.

Либо из терминала:
```bash
./gradlew assembleDebug
```
APK появится в `app/build/outputs/apk/debug/app-debug.apk`.

## Безопасность данных

- Паспортные данные хранятся через `EncryptedSharedPreferences` —
  ключ шифрования (AES-256-GCM) генерируется и хранится в **Android Keystore**,
  аппаратно защищённом модуле устройства. Сами данные никогда не попадают
  ни на какие сторонние серверы — только на `coko.tomsk.ru` по HTTPS.
- Кнопка "Выйти из аккаунта" в профиле полностью стирает сохранённые данные
  локально (и опционально может вызвать `CokoWebClient.logOff()` для
  завершения серверной сессии — сейчас это не подключено к UI, но метод готов).

## Известные ограничения / что доделать

- **Детализация по заданиям** — см. раздел выше, нужен реальный HTML
  открытой таблицы `#DetailsTable`.
- Расписание и результаты сейчас объединяются по совпадению (предмет, дата) —
  если на сайте оба раздела показывают предмет немного по-разному
  отформатированным, возможны дубликаты карточек на главном экране;
  легко поправить в `CokoRepository.loadDashboard()`.
- Статус "Скоро"/"Экзамен сдан" в `ExamStatus` для карточек из расписания
  сейчас всегда `UPCOMING` — сайт не отдаёт явного статуса для будущих
  экзаменов, только дату. Можно добавить сравнение даты с текущей.
- Форма обратной связи на экране "Помощь" не отправляет сообщение никуда —
  на сайте есть реальная страница `feedback.aspx`, её ещё не парсили.
- Апелляции на сайте — отдельная страница `appeals9.aspx`, сейчас в приложении
  это статичный список общих инструкций, не из самого сайта.
- Тёмная тема и уведомления в профиле — переключатели-заглушки на будущее.
